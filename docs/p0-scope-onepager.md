# MVP-001 — P0 Scope One-Pager

## Цель
Зафиксировать неизменяемый P0-контракт MVP для квиз-платформы: что обязательно должно работать к релизу и как измеряем готовность.

## Продуктовый результат P0
Пользователь может зайти в комнату, пройти квиз в реальном времени, увидеть результаты. Контент квиза загружается из заранее подготовленного набора вопросов.

## In-scope (P0 capabilities)

### P0-01. Lobby + join room
- Игрок вводит nickname и присоединяется к комнате по коду/ссылке.
- В комнате отображается список участников.
- Хост видит готовность игроков к старту.

**DoD:**
- Happy-path: игрок заходит в активную комнату и видит lobby ≤ 2 сек.
- Сервер валидирует уникальность nickname внутри комнаты.
- Ошибки join возвращаются пользователю с понятным текстом.

### P0-02. Запуск игровой сессии хостом
- Хост запускает игру из lobby.
- Создается game session с фиксированным набором вопросов.

**DoD:**
- Повторный клик Start не создает вторую сессию (idempotency).
- Все подключенные игроки переводятся в состояние вопроса синхронно.

### P0-03. Прохождение вопросов и отправка ответов
- Клиент показывает вопрос и варианты ответа.
- Игрок отправляет ответ в пределах таймера.

**DoD:**
- Принят только первый ответ игрока на вопрос.
- Ответы после таймера помечаются как late и не влияют на очки.
- Потеря ws-соединения не ломает сессию (возможен reconnect).

### P0-04. Подсчет очков и leaderboard
- Система начисляет очки по правилам pack.
- После каждого вопроса и в конце игры доступен рейтинг.

**DoD:**
- Баллы детерминированы и воспроизводимы для одинаковых входных данных.
- В финальном результате сохраняется топ и персональный результат игрока.

### P0-05. Контентный минимум
- Есть seed-данные: минимум 1 pack, 10+ вопросов, варианты/ответы, объяснение.
- Pack можно использовать в demo-сценарии end-to-end.

**DoD:**
- Миграции + seed разворачиваются на чистой БД без ручных правок.
- Вопросы валидны (без пустых prompt/options/correct_answer).

## Out of scope (P1/P2)
- Платежи и монетизация.
- Расширенная модерация UGC и антифрод.
- ML-персонализация контента.
- Полноценная BI/дашборды beyond MVP.
- Мультиязычность и локализация на уровне платформы.

## Нефункциональные ограничения P0
- Время восстановления backend при падении процесса: ≤ 5 мин (ручной рестарт допустим).
- Базовый мониторинг: health endpoint + error logging.
- Без требований к multi-region/high availability.

## Контрольные метрики выхода в MVP
- End-to-end demo (host + 2 players) проходит без ручных фиксов 3/3 раза.
- Ошибка join/start/submit < 1% на smoke-прогоне.
- 95-й перцентиль server response для критичных endpoint < 500ms локально.

## Владельцы и артефакты
- Product owner: scope + acceptance критерии.
- Tech lead: архитектура, контракты, state-machine.
- Backend/frontend owners: реализация и smoke-тесты.
