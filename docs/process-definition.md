# MVP-002 — Process Definition (DoR/DoD + tracking)

## 1. Цель
Определить единый процесс прохождения задач MVP: когда задачу можно брать в работу (DoR), когда считать завершенной (DoD), и как отражать статус.

## 2. Статусы задач
- **Not Started** — задача не начата.
- **In Progress** — есть активный исполнитель.
- **Blocked** — задача заблокирована внешней зависимостью.
- **Review** — код/док в ревью.
- **Done** — соответствует DoD и принята.

## 3. Definition of Ready (DoR)
Задачу можно брать в работу, если:
1. Есть привязка к MVP-* capability.
2. Описан ожидаемый результат (artifact/code behavior).
3. Прописаны acceptance criteria (минимум 2-3 проверяемых пункта).
4. Указаны зависимости и владелец.
5. Понятно, как демонстрировать результат.

## 4. Definition of Done (DoD)
Задача считается Done, если:
1. Реализация/документация соответствует acceptance criteria.
2. Обновлены связанные артефакты (если есть cross-impact).
3. Выполнены минимальные проверки:
   - lint/format (если применимо),
   - тесты/смоки (для runtime-фич),
   - миграции/seed (для DB-изменений).
4. Результат продемонстрирован на sync/demo.
5. Риски и ограничения зафиксированы в комментарии к задаче.

## 5. Формат weekly cadence
- **Daily 15m:** статус, блокеры, зависимости.
- **Design/API sync 2x week:** контракты, schema changes, state transitions.
- **Weekly demo:** проверка checkpoint-целей stage.
- **Weekly risk review:** топ-3 риска и mitigation.

## 6. Шаблон карточки задачи

```md
Title: MVP-XXX — <короткое имя>
Owner: <имя>
Stage: S0/S1/S2/...
Depends on: MVP-AAA, MVP-BBB

Context
- Почему задача нужна.

Acceptance criteria
- [ ] ...
- [ ] ...

Validation
- Команды/способ проверки.

Demo plan
- Что показать команде.

Risks
- Что может пойти не так.
```

## 7. Правила блокировок
- Любая блокировка > 24h эскалируется на daily.
- Если зависимость критична для пути MVP, создается отдельный blocker-ticket.
- В Blocked обязательно указывается owner unblock-действия.

## 8. Минимальный SLA по обновлению статусов
- Обновление статуса задачи: не реже 1 раза в день.
- In Progress без обновлений > 2 рабочих дней автоматически на review риска.
