# MVP-005 — Migration Gaps (draft)

Документ фиксирует разрыв между текущими миграциями и целевыми требованиями P0.

## Наблюдения по текущей схеме
- Базовые сущности для room/session/answers уже присутствуют.
- Есть поле `explanation` в вопросах, что покрывает контентный минимум.
- Нужна верификация ограничений уникальности для idempotent-поведения.

## Приоритетные gap-области

### GAP-1. Уникальность ответа игрока
**Проблема:** без уникального ограничения возможны дубли ответов.
**Действие:** добавить `UNIQUE(session_id, question_id, user_id)` в `answers`.
**Приоритет:** P0-blocker.

### GAP-2. Уникальность nickname в комнате
**Проблема:** дубли nickname ухудшают UX и лидеры могут конфликтовать.
**Действие:** добавить/проверить `UNIQUE(room_id, nickname)` в `room_members`.
**Приоритет:** P0-blocker.

### GAP-3. Индексы на leaderboard
**Проблема:** рост задержки на выдаче результатов при увеличении числа ответов.
**Действие:** индекс `session_scores(session_id, total_score DESC)`.
**Приоритет:** P1 (но желательно до тестового прогона нагрузки).

### GAP-4. Audit trace для state transition
**Проблема:** сложно разбирать инциденты по последовательности событий.
**Действие:** добавить lightweight `session_events` (опционально, если потребуется observability).
**Приоритет:** P1.
